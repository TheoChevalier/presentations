Définition

Un logiciel de gestion de versions permet de stocker un ensemble de fichiers en conservant la chronologie de toutes les modifications qui ont été effectuées dessus.

Centralisé Vs. Distribué

Mais la tendance…

Tout le monde a une copie de l'historique Le serveur facilite la transmission

Plusieurs logiciels

  • SVN (Subversion) - centralisé
  • HG (Mercurial) - distribué
  • GIT - distribué
SVN : Apache, sites web Mozilla... HG : Python, Thunderbird, Firefox ... GIT : VLC, Android, Firefox OS, Debian, Linux

BOUM.

Ok, comment j'annule ?

    Trois cas :
  1. Avant le commit
  2. Après le commit, mais avant le push
  3. Après le commit et le push

Nan, CTRL+Z, vous oubliez.

1. Avant le commit

            $ git checkout

    ou

            $ git checkout chemin/vers/monfichier
    checkout remet à zéro l'état de tous les fichiers

2. Après le commit, mais avant le push

        $ git reset --hard HEAD^

ou

        $ git reset --hard d6d9892
--hard /!\ Annule les commits et perd tous les changements HEAD^ peut être appellé plusieurs fois

3. Après le commit et le push

        $ git log
        $ git revert d6d9892
Créé un commit inverse hg appelle ça un backout // Repérer le numéro du commit à annuler

Mettre à jour le dépôt local

        $ git pull
s'assurer d'avoir tout commité Si conflit, il faut le corriger à la main faire un commit/push

Les branches

branche == laboratoire

N'influence pas le reste du projet Possible d'y travailler à plusieurs pouvoir tester plusieurs choses à la fois

Créer une branche

          $ git branch super_idee
Ne s'y déplace pas

Lister les branches

          $ git branch
          * master
            super_idee

Changer de branche

          $ git checkout super_idee
git modifie les fichiers à partir de là, faire des commits possible de créer une autre branche

Fusionner deux branches

          $ git merge super_idee

Exemple de conflit :

          <<<<<<< HEAD:file.txt
          Hello world
          =======
          Goodbye
          >>>>>>> 77976da35a11db4580b80ae27e8d65caf5208086:file.txt
sous-entendu vers master si il y a des conflits, éditer, puis commiter
